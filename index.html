PATH VISUALISER

Run different path finding algorithms on various mazes and topographies.

Get Started!

HOW TO USE

The path Visualiser software allows you to run various path finding algorithms, and the software will visualise the order in which the cells are explored, as well as the final, shortest path.

Click on any cell to toggle it to a wall, and click on a wall to remove it.

Select an algorithm and then click run to watch it traverse the grid.

OPTIONS

There are five options to toggle with when using the software:

  1. CLEAR BOARD: Clears every wall, topography and trail from the board, leaving only the start and end cells

  2. RESTORE BOARD: Restores the board to its initial state, preserving the start and end positions, the topographies and/or walls.

  3. RESET START AND END: Resets the start and end positions

  4. GENERATE TOPOGRAPHIES: Spawns random topographies, assigning each cell with a weight from 0-5. Higher weights equate to a higher cost of traversal to that cell.

  5. GENERATE MAZE: Generates a random maze for the algorithms to solve.

Algorithms

A brief look at each algorithm:

DEPTH FIRST SEARCH

DFS uses a stack to maintain the frontier, and so one an empty grid will snake back and forth before backtracking. On a maze it will explore the entirety of one corridor before backtracking upon reaching a dead end. It has no use of heuristics.

BREADTH FIRST SEARCH

BFS uses a queue to maintain the frontier. Every possible outgoing square from the frontier is expanded in each iteration. In a maze, it will traverse every corridor simultaneously. It has no use of heuristics.

A*

A* search uses heuristics to hone in on the goal. It keeps track of two values; the distance from the start node, and the distance to the goal, and chooses the cell with the smallest cost at each iteration. On a topography, it will chose the route of smallest cost (i.e avoiding the costly squares).

Dijkstra's

Maintains a priority queue of each cell, ordered by the shortest distance to get there. On a blank grid, will act like a BFS. On a topography, will find the route of smallest cost to the goal.

Lowest Cost First

Orders cells by the cost to reach them, not the distance. On a blank grid acts as a BFS as every cell has no cost associated with it. On a topography, finds the path of least cost, even if this may not be the shortest.

GREEDY BEST FIRST SEARCH

Uses heuristics to attempt to hone in on the goal. It has no regard for cost, for at each iteration it picks the cell closest to the goal in the hopes of reaching it quicker.
