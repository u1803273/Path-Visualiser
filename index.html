PATHFINDING VISUALISER

Next

This short tutorial will walk you through the features of this application



Select from one of the six algorithms:


  1. A*: Uses heuristics to hone in on the goal. Each node is given a cost; the sum of the distance left to travel to the goal + the cost of travelling to the current square. This incorporates costs from topographies.
    Guaranteed to find the optimal path
  2. Dijkstra: Works out the shortest cost of the path from the start to the goal. On an empty grid, it acts the same as a BFS, however, unlike a BFS, will take costs into account on topographies.
  3. BFS: Breadth First Search acts unintelligently, expanding all nodes from the start until it reaches the destination. It has no intuition of which path is the 'best' and which potentially leads closer to the goal. On a maze, it will expand along all corridors simultaneously.
  4. DFS: Depth First Search acts unintelligently, expanding all the nodes from the start until it reaches the destination. It similarly has no intuition of which path is best, but expands all along a single path until there exists no further unexplored squares, at which point it begins to backtrack. On a maze, it will expand all the way along a corridor until it reaches a dead end before it decides to backtrack and attempt a new route.
  5. Greedy: Greedy search order nodes via a heuristic, and always chooses the node closest to the goal. It is oblivious to cost, and so does not alter depending on topographies.
  6. Lowest Cost First: Lowest Cost first chooses squares based on their cost. If no topography exists it exhibits behaviour identical to BFS. However, on a topography graph, it will follow the path that leads to the goal with the lowest goal (thereby avoiding going over hills). It will find the path of least cost, but it may not be the shortest.

WALLS

Finish

Place walls onto the grid for the algorithm to avoid

Click a square to toggle it to a wall, and drag over the screen to place walls in a line. The algorithm will avoid the walls and find the shortest path while avoiding them.


Clicking on the mazes and patterns button will allow you to create mazes for which the algorithm will have to navigate. Algorithms such as BFS and DFS will handle these differently, with heuristic searches such as A* and Greedy Search attempting to hone in on the target without the need to examine every possible corridor.


The start and end nodes can be dragged to alter their positions.
